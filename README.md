## 内容策略

### 三大内容板块

```
博客
├── 日常（daily）   — 考研日志、学习进度、时间复盘
├── 思考（thinking） — 游戏拆解、影评、读后感
└── 工作（work）     — 科研阅读笔记、项目进度
```

### 各板块定位

| 板块 | 读者画像 | 内容调性 | 更新频率 |
|------|---------|---------|---------|
| **日常** | 真实、克制、有复盘感。不是碎碎念，是结构化的进度记录 | 每周 1-2 篇 |
| **思考** | 分析性 + 个人感受。拆解要结构化，感受要真诚 | 不定期 |
| **工作** | 学术但不死板。笔记要有自己的理解，不只是翻译摘要 | 每 1-2 周 1 篇 |

### 禁止事项

- ❌ 不写流水账式的情绪日记（日常板块不是朋友圈）
- ❌ 不堆砌术语炫技（导师比你懂得多，游戏公司看的是思维不是词库）
- ❌ 不追求更新频率而牺牲质量（宁可断更两周，不敷衍一篇）
- ❌ 不转载、不聚合、不水文（所有内容均为原创）

## 设计原则

1. **内容第一**：设计是退后的，让文字说话。不靠视觉元素吸引眼球。
2. **克制而非贫乏**：少即是多，但不等于简陋。排版舒适、配色稳妥、层级清晰。
3. **安静但有温度**：像一个收拾干净的书桌，不喧哗但有人情味。
4. **板块可识别**：三大内容方向之间读者能快速区分，但不是割裂的独立站点。
5. **零摩擦写作**：打开编辑器 → 写 Markdown → `git push`。不需要打开浏览器、不需要登录后台。

## 技术决策

| 决策 | 选择 | 理由 |
|------|------|------|
| **框架** | Hugo | 单二进制、毫秒构建、Go 生态、零维护 |

| **评论** | 暂不接入 | 短期内不需要 |
| **分析** | 暂不接入 | 短期内不需要 |
| **域名** | sumingwong.github.io | 暂用默认域名 |

## 写作规范（参考）

### 文章 frontmatter

```yaml
---
title: "文章标题"
date: 2026-06-12
categories: ["daily"]  # daily / thinking / work
tags: ["认知心理学", "注意", "实验设计"]
description: "一句话描述，用于首页摘要和 SEO"
draft: false
---
```

### 命名规范

- 文件命名：`YYYY-MM-DD-简短英文标题.md`
- 中文标题写在 frontmatter 的 `title` 字段
- 「简短英文标题」仅用于 URL，不需要完整翻译
