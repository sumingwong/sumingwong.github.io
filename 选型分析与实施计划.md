## 一、需求分析

### 1.1 当前背景

- 苏楚，武汉大学心理学 2026 届本科生，已从时代天使产品经理岗位离职，**全职备考心理学 312 统考（二战）**。
- 同时保持对游戏开发（Godot）、游戏策划、深度学习的兴趣和投入。

### 1.2 博客内容方向（三大板块）

| 板块 | 代号 | 内容 | 更新频率预期 |
|------|------|------|-------------|
| **日常** | `daily` | 考研日志、学习进度、时间安排复盘 | 日/周更 |
| **思考** | `thinking` | 游戏拆解文档、电影观后感、书籍读后感 | 不定期，取决于观影/阅读/游玩节奏 |
| **工作** | `work` | 科研论文阅读笔记、在手的项目进度 | 周/双周更 |

### 1.3 核心目的（双场景）

**场景 A：学术展示（考研复试 + 导师套辞）**
导师打开博客 → 看到苏楚对心理学研究的持续关注和深度思考 → 建立「这个学生对科研有真正的兴趣和投入」的印象。

**场景 B：职业铺垫（游戏策划 / 相关岗位）**
面试官打开博客 → 看到苏楚的游戏拆解能力和策划思维 → 建立「这个人玩游戏不是消遣，而是在思考设计」的印象。

### 1.4 设计约束

- **低维护成本**：备考期间时间紧张，博客不能成为负担。写作流程应为「打开编辑器 → 写 Markdown → git push」三步以内。
- **板块可区分**：三大内容板块应让读者能快速定位感兴趣的部分，而非混在一锅时间线里。
- **学术气质但不死板**：对导师要显得认真可靠，但不应像论文模板；对游戏公司要显得有审美和独立思考。
- **中文优先**：内容以中文为主。

## 二、框架选型

### 2.1 候选对比

| 维度 | Jekyll | Hugo | Astro | Hexo |
|------|--------|------|-------|------|
| **安装** | Ruby + gem（macOS 自带但版本可能偏旧） | `brew install hugo`，单二进制 | Node + 项目依赖 | Node + 项目依赖 |
| **构建速度** | 秒级（文章多了会变慢） | 毫秒级 | 快 | 秒级 |
| **GitHub Pages 部署** | 原生支持，push 即部署 | 需 GitHub Actions（一次配置） | 需 GitHub Actions | 需 GitHub Actions |
| **主题生态** | 丰富，偏文档/知识库风格 | 丰富，偏博客/写作风格 | 较新，主题少 | 中文社区多 |
| **学习曲线** | 低 | 低 | 中高 | 低 |
| **维护心智** | Ruby 版本升级偶尔踩坑 | 几乎零 | Node 依赖管理 | Node 依赖管理 |

### 2.2 结论：选择 Hugo

核心理由：

1. **安装零摩擦**：`brew install hugo`，没有依赖链问题。Jekyll 的 Ruby 版本管理在 macOS 上是个已知痛点。
2. **构建毫秒级**：即使将来上百篇文章，`hugo serve` 的实时预览也不会有任何延迟感。
3. **部署只配一次**：GitHub Actions 配置 3 个文件，之后每次 push 自动部署，体验与 Jekyll 原生支持无异。
4. **主题气质匹配**：Hugo 社区的博客主题设计哲学更偏向「让文字呼吸」，而非「让信息结构化」。

> 代价：相比 Jekyll 多了一次 GitHub Actions 的初始配置（约 10 分钟）。这个代价只付一次。

## 三、主题选型

### 3.1 候选对比

| 主题 | Stars | 气质 | 板块区分 | 暗色模式 | 图文混排 | 学术适配度 | 游戏拆解适配度 |
|------|-------|------|---------|---------|---------|-----------|-------------|
| **PaperMod** | ~10k | 安静、文字主导、像深夜书房 | 导航栏+标签 | ✅ 原生 | ✅ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ |
| Chirpy (Jekyll) | 9.9k | 技术文档/知识库，侧边栏分类 | 侧边栏分类 | ✅ | ✅ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |
| Stack | ~5k | 卡片式、视觉化 | 板块独立 | ✅ | ⭐⭐⭐⭐⭐ | ⭐⭐ | ⭐⭐⭐⭐⭐ |
| DoIt | ~1k | 功能全面、多语言 | 标签系统 | ✅ | ✅ | ⭐⭐⭐ | ⭐⭐⭐ |

### 3.2 结论：选择 Hugo + PaperMod

**为什么不是 Chirpy？**

Chirpy 的侧边栏分类结构对「学术展示」场景确实更直观（导师一眼看到分类），但它有两个问题：
- 框架上属于 Jekyll 生态，安装和维护成本更高。
- 设计语言偏「技术文档」，放在学术场景没问题，但放在「游戏拆解」场景会显得气质不合。

**为什么不是 Stack？**

Stack 的卡片式布局对游戏拆解文章（截图+分析）展示效果最好，但学术内容放在卡片里不够严肃，整体风格偏轻量。

**为什么是 PaperMod？**

PaperMod 的设计是「退后」的，它不替作者说话。打开博客看到的是干净的标题列表，点击进去是舒适的排版。这种克制在两个场景里都是加分项：

- **对导师**：干净、专注、不花哨 → 「这个学生做事踏实，有学术品味」
- **对游戏公司**：有审美、不跟风、独立思考 → 「游戏策划需要这种能静下心的人」

更重要的是，PaperMod 可以通过**分类导航**实现板块区分：导航栏设三个入口（日常 / 思考 / 工作），点击后展示对应分类的文章。这一点的实现很轻量，不需要改主题源码。

### 3.3 主题预览（心理图像）

- **首页**：深色背景上白色的文章标题列表，每一条标题下方附日期和分类标签。没有头像、没有动画、没有卡片阴影。像一个安静的个人图书馆目录。
- **文章页**：宽松的行距，适中的字重，左对齐的正文。代码块和引用块有克制的视觉区分。阅读时唯一在动的是向下滚动的进度条。
- **暗色模式**：默认。白色文字衬深灰底，阅读体验接近 Kindle。

## 四、信息架构

### 4.1 导航结构

```
首页 (/)
├── 日常 (/categories/daily/)      ← 考研日志与进度
├── 思考 (/categories/thinking/)   ← 游戏拆解、影评、读后感
├── 工作 (/categories/work/)       ← 科研笔记、项目进度
├── 关于 (/about/)                 ← 个人简介与博客说明
├── 标签 (/tags/)                  ← 全站标签索引
└── 搜索 (/search/)                ← 全文搜索
```

### 4.2 内容组织规则

- 每篇文章归属于**一个主分类**（daily / thinking / work）。
- 文章可额外打**标签**（如：认知心理学、Godot、黑神话、注意、实验设计）。
- 标签用于跨板块的横向关联（比如一篇关于游戏内注意力机制的拆解，同时归属「thinking」分类、打上「注意」「认知心理学」标签）。

### 4.3 URL 设计

```
文章：  /posts/2026-06-12-考研第一周复盘/
分类：  /categories/daily/
标签：  /tags/认知心理学/
关于：  /about/
```

## 五、实施计划

### Phase 1：环境初始化（预计 20 分钟）

| 步骤 | 内容 | 交付物 |
|------|------|--------|
| 1.1 | 安装 Hugo Extended | `brew install hugo` |
| 1.2 | 在仓库中初始化 Hugo 站点 | 站点骨架（config、content、themes 目录） |
| 1.3 | 安装 PaperMod 主题（Git submodule） | `themes/PaperMod/` |
| 1.4 | 配置 .gitignore | 忽略 `/public/`、`/.hugo_build.lock` |

### Phase 2：站点配置（预计 30 分钟）

| 步骤 | 内容 | 交付物 |
|------|------|--------|
| 2.1 | 编写 `hugo.toml`：站点名称、描述、语言、默认暗色模式 | `hugo.toml` |
| 2.2 | 配置导航菜单（日常/思考/工作/关于/标签/搜索） | 同 `hugo.toml` |
| 2.3 | 配置分类（taxonomies）：category + tag | 同 `hugo.toml` |
| 2.4 | 中文排版参数：字体、字号、行距、段落间距 | `assets/css/extended/custom.css` |

### Phase 3：主题定制（预计 20 分钟）

| 步骤 | 内容 | 交付物 |
|------|------|--------|
| 3.1 | 首页配置：显示摘要、不显示全文 | `hugo.toml` 中的 `params` |
| 3.2 | 配色微调（PaperMod 默认偏冷，往暖调拉） | `assets/css/extended/colors.css` |
| 3.3 | 头像与页脚信息 | `hugo.toml` 中的 `params` |
| 3.4 | 关于页面模板 | `content/about.md` |

### Phase 4：部署配置（预计 15 分钟）

| 步骤 | 内容 | 交付物 |
|------|------|--------|
| 4.1 | 编写 GitHub Actions 部署脚本 | `.github/workflows/hugo.yaml` |
| 4.2 | 配置 GitHub Pages source 为 GitHub Actions | 仓库 Settings → Pages |
| 4.3 | 推送并验证部署 | 访问 sumingwong.github.io 确认上线 |

### Phase 5：初始内容（预计 30 分钟）

| 步骤 | 内容 | 交付物 |
|------|------|--------|
| 5.1 | 关于页面初稿 | `content/about.md` 内容填充 |
| 5.2 | 第一篇占位文章（考研日志） | `content/posts/2026-06-12-hello-world.md` |
| 5.3 | 示例文章各一篇（展示三大板块） | 三篇草稿 |
